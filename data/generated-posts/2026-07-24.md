React had a 400ms hitch.

We were shipping a checkout flow in React.
The page looked fine in dev.
On a real phone, the button lagged just enough to feel broken.

We had spent 3 weeks polishing the design system.
New cards, animated drawers, prettier empty states.
None of it mattered when the first tap felt slow.

The bug was not one big thing.
It was 18 tiny rerenders from a parent component that owned too much state.
Every keystroke in one field forced half the page to repaint.

We fixed it by moving that state down and memoizing the expensive list.
React.memo means “skip work if props didn’t change.”
The lag dropped from 400ms to 40ms.

That was the whole lesson:
in React, pretty UI does not save bad state boundaries.

If a screen feels slow, I now ask one question first:
who owns this state, and why?

What state boundary have you seen quietly ruin a good interface?
