# LinkedIn Post

Topic: AI Agents

Date: 2026-07-14

---

This week’s agent passed 73 tests.

I’m building an AI assistant for support teams.
It reads messy customer threads and drafts replies for human review.

Last Monday, it was helping with 4 tickets a day.
By Friday, it was handling 31.

The thing I shipped this week was a tighter tool schema — the menu of actions the bot can take — plus better memory around customer history.

That moved draft approval rate from 58% to 81%.

But it also exposed a new mess:
the agent still hallucinates order numbers when a thread has screenshots and forwarded emails in the same conversation.

I spent two hours chasing one bad reply.
The root cause was simple: I trusted the model to “understand” the inbox layout.
It doesn’t.
It needs the layout spelled out every time.

That’s the lesson I keep relearning:
agents get better when the job is narrower, not when the prompt is longer.

What would you narrow first if your agent kept guessing wrong?
