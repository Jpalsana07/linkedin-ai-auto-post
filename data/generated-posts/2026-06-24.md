# LinkedIn Post

Topic: Agent Reliability

Date: 2026-06-24

---

Our agent sent 47 wrong emails.

We were building a support assistant to answer customer tickets and save the team time.
It had access to billing notes, order history, and our help docs.
One bad reply could confuse a real customer and create more work for support.

The bug was not the model.
It was trust.

We let the agent draft an email, then only checked if the text sounded polite.
It sounded great.
It also invented a shipping policy that did not exist.

The fix was boring, and it worked.
We made the agent quote the exact source line for every factual claim.
No source, no send.

After that, the false answers dropped fast.
In one week, the error rate fell from 18% to 2%.

The lesson: if an agent can act on the world, it needs proof, not confidence.

Where do you draw the line between letting an agent speak freely and making it show its work?
